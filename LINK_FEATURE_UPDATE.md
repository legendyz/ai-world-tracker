# 人工审核功能更新 - 网页链接支持

## 📅 更新日期
2025-12-04

## 🎯 更新内容

### 新增功能：网页链接查看

在人工审核过程中，现在可以直接在浏览器中打开内容的原文链接，以便更好地了解内容并做出准确判断。

---

## ✨ 主要改进

### 1. 自动显示链接
审核界面会自动检测并显示内容的网页链接（如果有）：

```
标题: OpenAI Launches GPT-5 with Enhanced Reasoning
来源: TechCrunch
🔗 链接: https://techcrunch.com/2025/openai-gpt5  ← 新增显示
摘要: OpenAI today announced GPT-5...
```

### 2. 新增操作选项
增加了选项 **6. 在浏览器中打开链接**：

```
操作选项:
  1. 保持当前分类
  2. 修改分类
  3. 修改技术领域
  4. 修改地区
  5. 标记为垃圾内容（删除）
  6. 在浏览器中打开链接  ← 新增！
  0. 跳过（稍后处理）
```

### 3. 灵活的字段支持
自动识别两种链接字段：
- `link` 字段（主要使用）
- `url` 字段（备选）

### 4. 无缝工作流程
```
选择操作 6 → 浏览器自动打开 → 查看原文 → 返回终端 → 继续选择操作
```

## 💡 使用场景

### 场景1：标题不清晰
**问题**：标题 "Company Announces Major Investment" 无法判断是产品还是市场新闻

**解决**：
1. 选择操作 6 打开链接
2. 阅读完整文章
3. 发现是融资新闻
4. 返回选择操作 2，修改为 `market` 分类

### 场景2：验证内容真实性
**问题**：标题 "AI Breakthrough Claims Questioned" 可能是假新闻

**解决**：
1. 选择操作 6 打开链接
2. 检查来源和内容
3. 确认是谣言或未证实消息
4. 返回选择操作 5，标记为垃圾

### 场景3：边界案例判断
**问题**：内容可能属于 `research` 或 `product`，不确定

**解决**：
1. 选择操作 6 打开链接
2. 阅读技术细节和商业信息
3. 判断主要侧重点
4. 返回选择正确的分类

## 🔧 技术细节

### 代码变更
**文件**: `manual_reviewer.py`

**变更1**: 显示链接
```python
# 显示链接（如果有）
link = item.get('link') or item.get('url')
if link:
    print(f"🔗 链接: {link}")
```

**变更2**: 新增操作选项
```python
if link:
    print("  6. 在浏览器中打开链接")
```

**变更3**: 处理链接打开
```python
elif choice == '6' and link:
    # 在浏览器中打开链接
    import webbrowser
    try:
        webbrowser.open(link)
        print(f"🌐 已在浏览器中打开链接")
        print("   查看完内容后，继续选择操作...")
        # 不返回，继续显示选项让用户做决定
    except Exception as e:
        print(f"❌ 无法打开链接: {e}")
```

## ❓ 常见问题

**Q: 所有内容都有链接吗？**
- A: 不是。只有数据采集时包含链接的内容才会显示。如果没有链接，选项6不会显示。

**Q: 打开链接后如何返回审核？**
- A: 查看完浏览器中的内容后，直接返回终端窗口，继续选择操作即可。

**Q: 链接打不开怎么办？**
- A: 系统会显示错误信息，你可以继续选择其他操作。可能原因：网络问题、链接失效、需要登录等。

**Q: 可以自定义浏览器吗？**
- A: 默认使用系统默认浏览器。如需指定浏览器，可以在系统设置中更改默认浏览器。

**Q: 对于 arXiv 论文如何处理？**
- A: arXiv 链接会正常打开，显示论文摘要页面。你可以在那里查看完整摘要和下载PDF。
